# docs

## ディレクトリの目的

本アプリケーションについての開発文書を管理する場所です。
コードから復元できない情報のみを文書として保持します（[ADR 021](adr/021_設計書のリポジトリ管理廃止.md)、[ADR 022](adr/022_docsの開発文書専用化.md)）。

## サブディレクトリ

### adr

アーキテクチャ決定記録（ADR）。技術的決定の経緯と理由を記録する。

### requirements

要件定義書。何を作るか・なぜ作るかを記録する。

### reports

調査報告書。技術検討の調査結果を記録する。

## 管理しないもの

- **設計書（機能設計書・機能仕様書・実装ガイド）**: コードとテストが動作する仕様である（[ADR 021](adr/021_設計書のリポジトリ管理廃止.md)）
- **サイト公開用コンテンツ原稿・執筆ガイドライン**: コンテンツはデータベースで管理し、MCP ツールで作成・更新する（[ADR 022](adr/022_docsの開発文書専用化.md)）
