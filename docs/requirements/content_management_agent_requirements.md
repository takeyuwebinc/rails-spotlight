# 掲載内容管理支援エージェント 要件定義書

## 1. 機能概要

spotlight-rails の掲載内容（Project / SpeakingEngagement / UsesItem / Slide）を、アプリケーション自身に組み込んだ AI エージェントとの対話で登録・更新できるようにする。

現状、掲載内容の更新は外部の AI エージェント（Claude Code 等）から MCP ツール経由で行っており、エージェントの起動・接続の手間がある。本機能により、管理画面のチャット UI だけで掲載内容の更新が完結する。また、自サイトへの AI エージェント組み込みは技術検証とポートフォリオ（技術力の提示）を兼ねる。

**主要な提供価値**：

- 管理画面のみで掲載内容の登録・更新が完結し、外部エージェントを起動する手間がなくなる
- 必須項目やタグ・スラッグの形式をエージェントが対話で引き出し整形するため、入力形式を覚えていなくても登録できる
- Rails アプリへの AI エージェント組み込みの実例として、技術検証・ポートフォリオの役割を果たす

## 2. 機能要件

### 2.1 基本処理フロー

1. **会話の開始・再開**
   - 管理者は管理画面のチャット UI で新しい会話を開始する、または会話一覧から過去の会話を選んで再開する
   - 会話履歴（完了した発言、ツール実行記録、LLM 利用トークン量・コスト）はデータベースに保存され、無期限に保持される
   - ブラウザを閉じた場合も、完了済みの発言までが保存され、続きから再開できる

2. **ヒアリングと素材収集**
   - 管理者は自然文で依頼する（例：「昨日の登壇を登録して」「このプロジェクトの説明を直したい」）
   - エージェントは登録・更新に必要な属性のうち不足しているものを質問して埋める。プレビュー提示の時点で必須属性が揃っていること
   - 管理者が URL を渡した場合、エージェントはそのページ内容を取得して属性を抽出する
   - エージェントは Web 検索によりイベント情報等の素材を収集できる

3. **承認なしで実行できる操作（読み取り系・外部取得系）**
   - 掲載内容の検索・一覧・参照（読み取り系ツール）、および URL 取得・Web 検索（外部取得系ツール）は、エージェントの判断で承認なしに実行する
   - 外部取得系ツールが取得した内容には悪意ある指示が混入し得る（プロンプトインジェクション）。取得内容は属性抽出の素材としてのみ扱い、データベースへの反映は承認ゲート（2.1.4）で防御する

4. **書き込み系操作（承認必須）**
   - 新規作成・更新・公開・非公開切替の実行前に、エージェントは変更内容のプレビューを提示する
   - 新規作成のプレビューには、作成時の公開状態（公開日時を含む）を管理者に確認した結果を含める
   - 管理者が承認した場合のみデータベースへ反映する。承認されたプレビューの内容と反映される内容は同一であること
   - 否認した場合は反映しない
   - 管理者はプレビューに対して修正指示ができ、エージェントは修正版のプレビューを再提示する
   - 承認後の反映時にバリデーションエラーが発生した場合、エージェントは修正内容を再度プレビューとして提示し、再承認を得てから反映する（承認なしの再実行はしない）

5. **応答表示とコスト表示**
   - エージェントの応答は生成中のテキストを逐次表示（ストリーミング）する。プレビューへの承認・否認・修正指示は、プレビューの表示が完了してから行う
   - 会話ごとに LLM 利用コストの概算を表示する。コストはさくらのAI Engine の公表単価に基づいて算出し、過去の会話を開いた場合も表示できる

### 2.2 対象データと操作範囲

| 対象 | 新規作成 | 更新 | 公開・非公開切替 | 削除 |
|------|---------|------|----------------|------|
| Project | ○ | ○ | ○ | 対象外 |
| SpeakingEngagement | ○ | ○ | ○ | 対象外 |
| UsesItem | ○ | ○ | ○ | 対象外 |
| Slide | ○ | ○ | ○ | 対象外 |

- 各モデルの検証規則（必須属性、スラッグの一意性）を満たすまでエージェントがヒアリングで補完する
- タグの付け替え（Slide / SpeakingEngagement）は更新に含む
- SpeakingEngagement は現状、個別に登録・更新する手段が存在しない（markdown 一括インポートのみ）。本機能で 4 種すべてを個別に操作できるようにする

### 2.3 モデル選択

- 会話進行、属性抽出、検索結果要約の各タスク種別ごとに、使用する LLM モデルを選択できる構成とする
- モデルの選択は開発者側の構成であり、管理者が画面上で選ぶ機能ではない
- タスク種別とモデルの具体的な割当は機能設計で決定する（要件では固定しない）
- ツール実行を伴うタスクには tool calling 対応モデルのみ割り当てる（制約 3.2 参照）

### 2.4 エラー処理

- **LLM API 呼び出し失敗**：エラーが発生したことを管理者に表示し、同じ内容を再送できる。会話履歴は失われない
- **コンテキスト長超過**：再送では解決しないため、新しい会話で続行するよう管理者に案内する
- **書き込み時のバリデーションエラー**：エージェントはエラー内容を踏まえて不足・不正の属性を管理者に確認し、修正版のプレビューを再提示する（2.1.4 の再承認フロー）
- **URL 取得・Web 検索の失敗**：失敗したことを会話内で伝え、管理者はテキスト貼り付けによる素材提供で続行できる

## 3. 前提条件・制約事項

### 3.1 前提条件

- 利用者は既存の admin 認証（Google OAuth）を通過した管理者本人のみ。権限分離は不要
- さくらのAI Engine のアカウントトークンと Brave Search API キーは Rails credentials に保存済み

### 3.2 制約事項

- **LLM プロバイダ**：さくらのAI Engine（OpenAI 互換 Chat Completions API、`https://api.ai.sakura.ad.jp/v1`）を使用する
- **tool calling 制約**：提供モデルのうち llm-jp-3.1-8x13b-instruct4 は tool calling 非対応（2026-07-20 実測。基盤側構成による制限）。gpt-oss-120b / Qwen3-Coder-480B-A35B-Instruct-FP8 / Qwen3-Coder-30B-A3B-Instruct は tool calling・マルチターン・ストリーミングの動作を実測確認済み
- **コスト算出制約**：ライブラリのモデルレジストリにさくらのAI Engine の単価情報はないため（実測で欠落または他社単価が混入）、コスト表示はさくらの公表単価を自前で保持して算出する。表示は概算とし、端数処理の詳細は機能設計で決定する
- **Web 検索**：Brave Search API を使用する
- **エージェント基盤**：ruby_llm gem（1.16 以上）および RubyLLM::Agent を使用する
- **コンテキスト長**：さくらのAI Engine の各モデルは 128K トークン制限（超過時はリクエストエラー）

### 3.3 依存関係

- 既存の MCP ツール群（`app/mcp/tools/`）と対象モデルは共通。本機能のエージェント用ツールの実装形態（既存 MCP ツールの再利用可否を含む）は機能設計で決定する

## 4. 監査・記録要件

利用者が管理者本人のみであるため、独立した監査ログは設けない。以下の整理とする。

- **記録の目的**：更新経緯の事後確認（「いつ・何を・どう更新したか」）
- **記録の手段**：保存される会話履歴（発言およびツール実行記録）をもって充足とする
- **保存期間**：無期限（会話の削除機能は本リリースでは提供しない。8 章参照）
- **参照者**：管理者本人のみ

## 5. リリース・運用要件

### 5.1 環境構築要件

- **シークレット**：さくらのAI Engine アカウントトークン、Brave Search API キー（Rails credentials に保存済みのため追加作業なし。CI・テスト環境では実際の API を呼ばないこと）
- **外部サービス**：さくらのAI Engine（契約済み）、Brave Search API（キー取得済み）
- **データベース**：会話履歴（発言、ツール実行記録、トークン量・コスト）の永続化のためのデータベース変更を伴う（リリース時に適用）

### 5.2 運用

- **単価の更新**：さくらのAI Engine の公表単価が改定された場合、自前で保持する単価を更新する。過去会話のコストを遡って再計算するかは問わない（表示は概算のため）

既存データの補正・移行、ユーザー通知、段階的ロールアウト、監視・アラート追加、定期バッチは不要（ヒアリングで確認済み）。

## 6. 用語定義

| 用語 | 説明 |
|------|------|
| 管理者 | admin 認証を通過した本サイトの所有者本人。本機能の唯一の利用者 |
| 掲載内容 | サイトに公開する 4 種のコンテンツ（Project / SpeakingEngagement / UsesItem / Slide）の総称 |
| エージェント | 本機能で spotlight-rails に組み込む、ツール実行能力を持つ LLM ベースの対話プログラム |
| 会話 | 管理者とエージェントの一連のやり取りの単位。データベースに保存され、一覧から再開できる |
| ツール | エージェントが呼び出せる機能単位。本文書では読み取り系・外部取得系・書き込み系の 3 区分で扱う（既存の MCP ツールとは区別する。実装上の関係は機能設計で決定） |
| 読み取り系ツール | 掲載内容を検索・一覧・参照するツール。データベースを変更しない |
| 外部取得系ツール | URL 取得・Web 検索によりアプリ外部から情報を取得するツール。データベースを変更しない |
| 書き込み系ツール | 掲載内容を作成・更新・公開・非公開切替するツール |
| プレビュー | 書き込み系ツールの実行前にエージェントが提示する変更内容の提案。受け入れ条件の「登録案」もこれを指す |
| 承認 | プレビューを確認した管理者が実行を許可する操作 |
| さくらのAI Engine | さくらインターネットが提供する OpenAI 互換の推論 API 基盤。本機能の LLM プロバイダ |

## 7. 受け入れ条件

**基本フロー**

- [ ] 管理者が管理画面から会話を開始し、対話で SpeakingEngagement を新規登録できる（必須項目が対話で補完される）
- [ ] Project / UsesItem / Slide についても同様に新規登録できる
- [ ] 既存レコードの属性を対話で更新できる（Slide / SpeakingEngagement のタグ付け替えを含む）
- [ ] 公開・非公開切替を対話で実行できる
- [ ] 新規作成のプレビューに公開状態（公開日時を含む）が含まれ、管理者への確認を経ている

**素材収集**

- [ ] イベントページの URL を渡すと、内容が抽出されてプレビューに反映される
- [ ] Web 検索で収集した素材がプレビューに反映される
- [ ] URL 取得・Web 検索が失敗した場合、その旨が会話に表示され、テキスト貼り付けで続行できる

**承認ゲート**

- [ ] 書き込み系操作の前に必ずプレビューが提示され、承認するまでデータベースが変更されない
- [ ] 承認されたプレビューの内容とデータベースへの反映内容が一致する
- [ ] プレビューへの修正指示で修正版プレビューが再提示される
- [ ] 否認した場合、データベースが変更されない
- [ ] 反映時にバリデーションエラーが発生した場合、修正版プレビューの再承認を経てから反映される
- [ ] 読み取り系・外部取得系ツール（例：既存スライドの一覧、Web 検索）は承認なしで実行される

**会話・表示**

- [ ] ブラウザを閉じた後、会話一覧から過去の会話を開いて続きから再開できる
- [ ] 応答がストリーミング表示される
- [ ] 会話画面に当該会話の LLM 利用コスト概算が表示され、さくらのAI Engine の公表単価に基づいている。過去の会話を開いた場合も表示される

**認証・エラー**

- [ ] admin 認証を通過していないアクセスではチャット UI・エージェント機能を利用できない
- [ ] LLM API 呼び出しが失敗した場合にエラーが表示され、再送で続行できる

## 8. スコープ外と今後の拡張

### ファーストリリース（今回対象）

- 2 章の機能要件のすべて

### 今回は対象外（セカンドリリース以降の候補）

- 掲載内容レコードの削除操作
- 会話の削除操作
- エージェント側からの更新候補の提案（棚卸し提案）
- 複数管理ユーザー対応
