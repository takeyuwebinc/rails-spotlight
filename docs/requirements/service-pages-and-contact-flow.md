# サービス案内ページ・問い合わせ導線 要件定義書

**作成日**: 2026年2月16日
**更新日**: 2026年2月16日

## 1. 機能概要

コーポレートサイトにサービス案内ページを新設し、サービス内容の紹介から問い合わせ（Google Forms）への導線を整備する機能。

現在のサイトにはサービス案内ページが存在せず、`/services` は `/projects` にリダイレクトされている。旧コーポレートサイト（rails-takeyuwebinc）にあったサービス案内を、一人会社の体制に合わせてリライトした上で移植する。

**想定閲覧者**：エンジニアリングマネージャー（開発委託先・技術支援先を探している）

**主要な提供価値**：
- 提供サービス（受託開発・技術顧問）の内容を訪問者に伝えられる
- サービス案内から問い合わせフォーム（Google Forms）への明確な導線ができる
- サイトのナビゲーションを再構成し、想定閲覧者にとって有効な情報提示順にする

## 2. 機能要件

### 2.1 サービス一覧ページ（`/services`）

1. **サービス一覧の表示**
   - 提供中のサービスを一覧表示する
   - 対象サービス：
     - 受託開発（Outsourcing）
     - 技術顧問（Technical Advisor）
   - 各サービスについて、概要と詳細ページへのリンクを表示する

2. **問い合わせへの導線**
   - ページ内にGoogle Formsへのリンク（CTA）を設置する

### 2.2 受託開発サービス詳細ページ（`/services/outsourcing`）

1. **サービス内容の表示**
   - 旧サイトの受託開発ページの内容を、一人会社の体制に合わせてリライトして掲載する
   - 旧サイトで紹介していた主な内容：
     - 新規Webサービス開発
     - 既存サービスの保守・レガシーシステム対応
     - 契約形態（準委任契約ベースの月額稼働 / 請負契約）
   - 一人会社としての強み・特徴が伝わる内容にする

2. **問い合わせへの導線**
   - ページ内にGoogle Formsへのリンク（CTA）を設置する

### 2.3 技術顧問サービス詳細ページ（`/services/technical_advisor`）

1. **サービス内容の表示**
   - 旧サイトの技術顧問ページの内容を、一人会社の体制に合わせてリライトして掲載する
   - 旧サイトで紹介していた主な内容：
     - 月額契約での技術支援
     - Ruby on Railsの専門知識によるサポート
     - 技術的負債の予防
     - オンライン・対面での相談対応
   - 一人会社としての強み・特徴が伝わる内容にする

2. **問い合わせへの導線**
   - ページ内にGoogle Formsへのリンク（CTA）を設置する

### 2.4 ナビゲーションの再構成

1. **ヘッダーナビゲーション**
   - 現在のメニュー構成（About / Articles / Projects / Speaking / Uses）を見直し、サービスを含めた構成に再編する
   - 想定閲覧者（エンジニアリングマネージャー）が委託先を検討する際の情報優先度に基づき配置する
     - まずサービス内容（何を提供するか）
     - 次に実績（信頼できるか）
     - 最後に人物・補足情報
   - 変更後のナビゲーション構成：**Services / Projects / Speaking / About / Uses**
   - Articlesは外部リンク（Zenn）のため、ナビゲーションから外しAboutページ内のリンクとする
   - デスクトップ・モバイル両方のナビゲーションを対象とする

2. **既存リダイレクトの更新**
   - 現在 `/services` → `/projects` へのリダイレクトが設定されているため、これを解除してサービス一覧ページを表示するように変更する
   - `/contacts` → `/about` のリダイレクトはそのまま維持する（問い合わせはGoogle Formsを使用するため）

### 2.5 既存ページの問い合わせ導線

- ホームページおよびAboutページに既に存在するGoogle Formsへのリンクは維持する

## 3. 前提条件・制約事項

### 3.1 前提条件

- 問い合わせフォームは既存のGoogle Forms（`https://forms.gle/scwNEGrT196rFnD9A`）を継続利用する
- サービスページのデザインは、既存のサイトのデザインシステム（Tailwind CSS、ダークモード対応）に準拠する
- サービスの説明文は旧サイトの内容をベースに、実装フェーズで一人会社向けにリライト案を作成する

### 3.2 制約事項

- **技術的制約**：既存のRails 8.0 / Tailwind CSSの構成に従う
- **コンテンツ制約**：旧サイトの「リモートチーム」サービスは終了済みのため移植対象外

### 3.3 依存関係

- 旧サイト（rails-takeyuwebinc）のサービス説明文をリライトのベースとして参照する

## 4. 用語定義

| 用語 | 説明 |
|------|------|
| 受託開発 | クライアントからの委託を受けてWebアプリケーションを開発するサービス |
| 技術顧問 | 月額契約でRails開発に関する技術的な相談・支援を行うサービス |
| CTA | Call To Action。ユーザーに問い合わせなどの行動を促すボタンやリンク |
| Google Forms | 問い合わせフォームとして使用している外部サービス |

## 5. 受け入れ条件

- [ ] `/services` でサービス一覧ページが表示されること
- [ ] `/services/outsourcing` で受託開発の詳細ページが表示されること
- [ ] `/services/technical_advisor` で技術顧問の詳細ページが表示されること
- [ ] 各サービスページからGoogle Formsへのリンクが機能すること
- [ ] ヘッダーナビゲーション（デスクトップ・モバイル）にサービスへの導線があること
- [ ] 既存ページ（ホーム・About）の問い合わせ導線が維持されていること
- [ ] ダークモードで正常に表示されること
- [ ] レスポンシブ対応（モバイル・デスクトップ）が機能すること
