# OAuth認証対応 要件定義書

## 1. 機能概要

本機能は、spotlight-railsアプリケーションに2つのOAuth認証機能を追加する。

1. **Admin画面のGoogleログイン**: 管理画面へのアクセスをGoogleアカウント認証に変更し、特定ドメインのユーザーのみアクセスを許可する
2. **MCPサーバーのOAuth認証**: Claude AIアプリからMCPサーバーを利用可能にするため、MCP仕様に準拠したOAuth認証を実装する

### 主要な提供価値

- Admin画面へのセキュアなアクセス管理（ドメイン制限付き）
- Claude AIアプリからのMCPサーバー利用の実現
- シングルサインオン（SSO）による利便性向上

## 2. 機能要件

### 2.1 Admin画面のGoogleログイン

#### 2.1.1 認証フロー

1. **ログインページ表示**
   - Admin画面にアクセスした未認証ユーザーにログインページを表示
   - 「Googleでログイン」ボタンを提供

2. **Google OAuth認証**
   - GoogleのOAuth 2.0認証フローを実行
   - ユーザーのメールアドレスとプロフィール情報を取得

3. **ドメイン検証**
   - 取得したメールアドレスのドメインが `@takeyuweb.co.jp` であることを検証
   - 許可されたドメイン以外のアカウントはアクセスを拒否

4. **セッション管理**
   - 認証成功時にRailsのセッション機能でログイン状態を保持
   - ログアウト機能を提供

#### 2.1.2 ユーザー管理

- ユーザーの事前登録は不要
- `@takeyuweb.co.jp` ドメインのGoogleアカウントであれば認証を許可
- ユーザー情報の永続化（データベース保存）は行わない

#### 2.1.3 セッション仕様

- Railsの標準セッション機能（Cookieベース）を使用
- セッション有効期限の特別な指定なし（Rails標準の動作に従う）

### 2.2 MCPサーバーのOAuth認証

#### 2.2.1 認証方式

- MCP（Model Context Protocol）仕様に準拠したOAuth 2.0認証を実装
- Claude AIアプリがOAuthクライアントとしてアクセス可能にする

#### 2.2.2 認証フロー

1. **認可リクエスト**
   - Claude AIアプリからの認可リクエストを受け付け
   - Googleアカウントでの認証を要求

2. **ユーザー認証**
   - Google OAuth認証を実行
   - `@takeyuweb.co.jp` ドメインのみ許可

3. **アクセストークン発行**
   - 認証成功時にアクセストークンを発行
   - トークンをClaude AIアプリに返却

4. **API認可**
   - MCPサーバーへのリクエスト時にアクセストークンを検証
   - 有効なトークンであればMCPツールへのアクセスを許可

#### 2.2.3 認可範囲

- MCPサーバーで提供しているすべてのツールへのアクセスを許可
- ツール単位のアクセス制御は行わない

#### 2.2.4 MCP仕様準拠

- MCP仕様で定義されたOAuthフローに準拠
- 必要なエンドポイント（認可、トークン発行等）を実装

## 3. 前提条件・制約事項

### 3.1 前提条件

- Google Cloud Platformでのプロジェクト設定が完了していること
- OAuth 2.0クライアントID/シークレットが取得済みであること
- 本番環境ではHTTPS通信が必須

### 3.2 制約事項

#### 技術的制約

- Rails 8.0.1環境での実装
- 既存のセッション管理（Cookieベース）を活用

#### 業務上の制約

- 許可ドメイン: `@takeyuweb.co.jp` のみ
- ユーザー情報のデータベース保存は行わない

### 3.3 依存関係

- Google OAuth 2.0 API
- MCP仕様（Model Context Protocol）
- mcp gem 0.4.0以上（現行0.1.0からアップデート）

### 3.4 トークン管理仕様

| 項目 | 値 |
|------|-----|
| アクセストークン有効期限 | 1時間 |
| リフレッシュトークン | 有効 |
| リフレッシュトークン有効期限 | 30日 |

## 4. 用語定義

| 用語 | 説明 |
|------|------|
| OAuth 2.0 | 認可フレームワーク。サードパーティアプリケーションがユーザーの代わりにリソースにアクセスするための仕組み |
| MCP | Model Context Protocol。AIモデルがツールやリソースにアクセスするための標準プロトコル |
| ドメイン制限 | 特定のメールドメイン（@takeyuweb.co.jp）を持つユーザーのみ認証を許可する制約 |
| アクセストークン | APIへのアクセスを認可するための認証情報 |
| Claude AIアプリ | AnthropicのClaude AIを利用するクライアントアプリケーション |

## 5. 受け入れ条件

### Admin画面のGoogleログイン

- [ ] Admin画面にアクセスするとGoogleログイン画面が表示される
- [ ] `@takeyuweb.co.jp` ドメインのGoogleアカウントでログインできる
- [ ] `@takeyuweb.co.jp` 以外のドメインのアカウントはログインが拒否される
- [ ] ログイン後、Admin画面の各機能にアクセスできる
- [ ] ログアウト機能が動作する

### MCPサーバーのOAuth認証

- [ ] Claude AIアプリからOAuth認証フローを開始できる
- [ ] `@takeyuweb.co.jp` ドメインのGoogleアカウントで認証できる
- [ ] `@takeyuweb.co.jp` 以外のドメインのアカウントは認証が拒否される
- [ ] 認証成功後、Claude AIアプリからMCPツールを利用できる
- [ ] MCP仕様に準拠したエンドポイントが実装されている

## 6. 現状からの変更点

### Admin画面

| 項目 | 現状 | 変更後 |
|------|------|--------|
| 認証方式 | HTTP Basic認証 | Google OAuth 2.0 |
| 認証情報管理 | Rails credentials | Googleアカウント |
| セッション管理 | ステートレス | Railsセッション（Cookie） |
| ドメイン制限 | なし | `@takeyuweb.co.jp` のみ |

### MCPサーバー

| 項目 | 現状 | 変更後 |
|------|------|--------|
| 認証方式 | Bearer Token（静的） | OAuth 2.0 |
| トークン管理 | Rails credentials | 動的トークン発行 |
| クライアント | 任意（トークン所持者） | Claude AIアプリ |
| ドメイン制限 | なし | `@takeyuweb.co.jp` のみ |
