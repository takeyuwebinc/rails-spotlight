# 竹内さんらしい文体ガイド

ポートフォリオサイトのコンテンツ作成時に参照する文体ガイドラインです。

## 1. 避けるべき表現（AIっぽい・営業トーク調）

| NG表現 | 問題点 |
|--------|--------|
| 「〜に情熱を持つ」 | 紋切り型、人柄が見えない |
| 「最適なソリューションを提供」 | 営業パンフレット調 |
| 「効率的に提供」「高品質な」 | 抽象的、セールストーク感 |
| 「お客様のビジネス成長をサポート」 | 大げさ、よそ行き |
| 「スケーラブルで効率的な」 | カタカナ形容詞の羅列 |
| 「クリエイティブな仕事をサポート」 | わざとらしい |
| 長い一文で複数の概念を詰め込む | 読みにくい、人工的 |

## 2. 使うべき表現・トーン

### ゆるさ・親しみやすさ
- 「ゆるく」「気軽に」「お気楽に」
- 「〜してみてください」「〜かもしれません」
- 断定を避け、柔らかく

### 謙虚さ・率直さ
- 「〜だと思います」「〜な気がします」
- 「うろ覚えですが」「僕の観測範囲では」
- 知識の限界を正直に認める

### 日常的な比喩・共感
- 「確定申告と同じで、貯めるとつらい」
- 「一人だと集中できない、飽きちゃう」
- 読者が「あるある」と思える表現

### シンプル・短文
- 一文一意
- 「〜です。〜しています。」と区切る
- 体言止めも効果的

## 3. 書き換え例

### Before（AI調）
> 複雑な専門知識や大規模チームを必要とせず、高品質なWebアプリケーションを効率的に提供。Railsの哲学を活かし、不必要な複雑さを排除することで、保守性が高く、長期的に進化し続けるシステムを構築しています。

### After（竹内さん調）
> 大きなチームや複雑な構成がなくても、Railsならちゃんと動くものが作れる。そのシンプルさが気に入っています。

---

### Before
> 反復的なタスクを削減し、本質的な問題解決と設計に集中。この内部効率化によって、より迅速かつ高品質なサービス提供を実現し、お客様のビジネス成長を技術面から長期的にサポートしています。

### After
> 地味な作業を手伝ってもらえるのがありがたいですね。その分、設計や問題解決にじっくり向き合えるようになりました。

---

### Before
> お客様のビジネスに合わせた最適なソリューションを提供しています。

### After
> （削除、または）よかったら見ていってください。

## 4. チェックリスト

文章を書いたら確認：

- [ ] 「情熱」「最適」「効率的」「高品質」などの抽象語を使っていないか
- [ ] 一文が長すぎないか（40文字以上なら分割を検討）
- [ ] 「〜を提供」「〜をサポート」で終わっていないか
- [ ] 読んで「営業資料っぽい」と感じないか
- [ ] 自分が話し言葉で言いそうな表現か
- [ ] 謙虚さ・ゆるさがあるか
